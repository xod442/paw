Everything starts with the Inventory

Assumptions

The switches supported currently are ones that utilize BGP un-numbered interfaces.

The names of the leaf hosts follows the format of (POD name).(switch_role). EX. pod2.spine01 The hostnames, on the devices themselves, the host name MUST match the leaf host name. EX pod2.spine01. ANY CHANGE TO THIS WILL CAUSE THIS APPLICATION TO FAIL!!!!!!

Leaf is a mongo database used for managing variables, playbooks, inventoryand templates for managing spine/leaf networks with Ansible. The fields in the database hold the minimal variables needed to spawn all the information needed be managed by ansible. After performing the Zero Thouch operation, the switches will be ready for ansible-playbooks.

Using a zero touch provisioning techops process with a Linux host configured with DHCP option 67 truly provides for no touch (of the switch!).
Switches load a python file and execute it locally and create the basic configuration necessary for Ansible management.
Once the ZTP process has ran, the switches are ready to be managed by the playbooks used with this program.
Current Behavior
This application uses "SEED" variables. You enter the seed IP addresses and the system will generate all the ALL.YAML variables required for the Jinja2 templates. At this release the primary fileds are as follows.
POD

The name of the POD. Used to track different data center installations. (POD1, DC2-POD2, etc)


Router ID

Seed address for router id's
